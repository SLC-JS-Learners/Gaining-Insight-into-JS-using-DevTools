Dev Tools Intro

  • currently Chrome has the best dev tools for this demo
  • We'll be working mostly with a combination of the "Sources" tab with the "Console" output visible:
    1. Keyboard open "Console" command in Chrome
      • Mac: cmd-opt-j / ⌘⌥j
      • Windows: ctrl-shift-j
    2. cmd + ] to shift right from the "Console" tab to the "Sources" tab
    3. esc to toggle the "Console" view in a Dev Tools tab other than the "Console" tab
  • We need to tell Chrome where our local directory is, and give it permission to make modifications:
    1. Click the "Filesystem" tab.
    2. Click the "+ Add folder to workspace" button.
    3. Navigate down to your cloned directory of the repo.
    4. Click the "Select" button.
    5. In the yellow toaster, click the "Allow" button.
    6. Ensure you're showing the "index.html" file's contents.
    7. Hide the Navigator sidebar.
    8. /// - expand/collapse panels
  • cmd-s - saves your file
  • cmd-r - refreshes your webpage

Debugging Intro

  • debugger - code execution is paused at the debugger statement
  • ⌘\ resumes script
  • ⌘; runs the next executable statement
  • ⌘⇧ run the remaining executable statements in the current function, then stop at the next executable statement outside of this function
  • ⌘' run the next function and then stop at the next executable function
debugger;

console.log('Hello World!');

Scope - Global

debugger;

function aFunction () {
	console.log('Hello World from aFunction');
};

var aVar = 'Hello World from aVar';

let aLet = 'Hello World from aLet';

const aConst = 'Hello World from aConst';
  • scope - the current context of the code
    (e.g. "Lady Gaga" is recognized globally,
    "fry sauce" is recognized within Utah,
    "Sean" is recognized with the Utah Devs at Frontline Education)
  • hoisting "...variable and function declarations are put into memory during the compile phase, but stays exactly where you typed it in your coding."
  • undefined - (a) A variable that exists but has not been assigned a value yet, or (b) the return value of a function without an assigned return value.
  • temporal dead zone / "not defined" - let & const must be initialized before they can be used.

Scope - Local

debugger;

function aGreeting (greeting, greetingName) {
	return function () {
		const msg = greeting + ' ' + greetingName + '!';
		return msg
	}
};

const aGreet = (greetName) => {
	return aGreeting('Hello', greetName)
}

let aGreetWorld = aGreet('World');

var aWholeGreeting = aGreetWorld();

console.log(aWholeGreeting);
